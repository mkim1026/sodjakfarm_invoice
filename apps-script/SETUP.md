# 직원 보고 시스템 – 설정 가이드

## 개요

이 시스템은 다음 구성 요소로 이루어집니다.

```
[직원이 index.html 폼 작성]
         │ HTTP POST
         ▼
[Google Apps Script Web App]
         ├──▶ Google Sheets에 행 추가/수정/삭제
         └──▶ 텔레그램 봇 → 그룹 채팅 알림
```

---

## STEP 1. 구글 시트 생성

1. [Google Sheets](https://sheets.google.com) 접속 후 **새 스프레드시트** 생성
2. 이름 예시: `직원 보고 2026`
3. 브라우저 주소창에서 **Sheet ID** 복사
   - URL 예: `https://docs.google.com/spreadsheets/d/`**`1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms`**`/edit`
   - 굵게 표시된 부분이 Sheet ID

---

## STEP 2. Apps Script 설정

1. 해당 스프레드시트에서 상단 메뉴 → **확장 프로그램 → Apps Script** 클릭
2. 에디터에 있는 기본 코드를 모두 지우고 `Code.gs` 파일의 내용을 붙여넣기
3. 상단의 설정값 수정:

```javascript
const SHEET_ID       = '여기에_시트_ID_붙여넣기';
const SHEET_NAME     = '직원보고';           // 탭 이름 (변경 가능)
const TELEGRAM_TOKEN = '여기에_봇_토큰';
const TELEGRAM_CHAT_ID = '여기에_채팅_ID';
```

4. **저장** (Ctrl+S / Cmd+S)

---

## STEP 3. Web App 배포

1. 우측 상단 **배포 → 새 배포** 클릭
2. 왼쪽 유형에서 **웹 앱** 선택
3. 아래와 같이 설정:
   - 설명: `직원보고 API v1`
   - 실행 계정: **나 (내 계정)**
   - 액세스 권한: **모든 사용자**
4. **배포** 클릭
5. Google 계정 권한 요청 시 허용
6. 생성된 **웹 앱 URL** 복사 (형태: `https://script.google.com/macros/s/.../exec`)

> ⚠️ 코드를 수정할 때마다 **새 배포** 또는 **기존 배포 업데이트**를 해야 변경사항이 반영됩니다.

---

## STEP 4. 텔레그램 봇 설정

### 4-1. 봇 생성

1. 텔레그램에서 **@BotFather** 검색 후 대화 시작
2. `/newbot` 명령 전송
3. 봇 이름과 username 입력 (username은 `bot`으로 끝나야 함, 예: `sodjakfarm_bot`)
4. BotFather가 제공하는 **API Token** 복사
   - 형태: `7123456789:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

### 4-2. 그룹 채팅에 봇 추가

1. 알림을 받을 텔레그램 그룹(채널)에 봇을 초대
2. 그룹에서 아무 메시지나 전송

### 4-3. Chat ID 확인

브라우저에서 아래 URL 접속 (토큰 교체):

```
https://api.telegram.org/bot[YOUR_TOKEN]/getUpdates
```

응답 JSON에서 `"chat":{"id":` 뒤의 숫자 확인
- 개인 채팅: 양수 (예: `123456789`)
- 그룹 채팅: 음수 (예: `-1001234567890`)

---

## STEP 5. index.html 연동

1. 앱에서 **직원 보고** 메뉴 클릭
2. 상단 설정란에 STEP 3에서 복사한 **Web App URL** 붙여넣기
3. URL은 브라우저 localStorage에 자동 저장됨

---

## 구글 시트 컬럼 구조

| 열  | 내용            |
|-----|----------------|
| A   | ID (자동생성)   |
| B   | 이름            |
| C   | 이메일          |
| D   | 날짜            |
| E   | 유형 (연차/반차/외근) |
| F   | 사유            |
| G   | 외근지          |
| H   | 제출시간        |
| I   | 마지막수정시간  |

> 시트는 앱 최초 제출 시 자동으로 생성되고 헤더도 자동으로 추가됩니다.

---

## 수정/삭제 인증 방식

- 직원이 보고서를 제출할 때 이메일을 함께 입력
- 수정/삭제 시 같은 이메일을 다시 입력해야 처리 가능
- Apps Script 서버 측에서 이메일 일치 여부를 검증 (클라이언트 조작 불가)

---

## 텔레그램 알림 형식

```
🔔 [직원 보고 - 신규]
👤 이름: 홍길동
📋 유형: 외근
📅 날짜: 2026-04-11
📍 사유: 거래처 방문
🏢 외근지: 비엔티안 센터
⏰ 제출: 2026-04-11 14:30
```

수정 시 `✏️ [직원 보고 - 수정]`, 삭제 시 `🗑️ [직원 보고 - 삭제]` 메시지 전송

---

## 문제 해결

| 증상 | 원인 및 해결 |
|------|-------------|
| "연결 오류" 메시지 | Apps Script URL이 잘못됐거나 배포가 안 됨 → URL 재확인 |
| 시트에 데이터가 없음 | SHEET_ID가 틀렸거나 권한 없음 → 시트 공유 설정 확인 |
| 텔레그램 알림이 안 옴 | TOKEN 또는 CHAT_ID 오타 → BotFather에서 재확인 |
| "이메일이 일치하지 않습니다" | 대소문자 차이 없음 (소문자로 비교) → 제출 시 이메일 정확히 입력 |
| 코드 수정 후 변경 없음 | 배포 업데이트를 안 함 → 배포 → 기존 배포 관리 → 편집 → 저장 |
